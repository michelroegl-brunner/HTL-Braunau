\chapter{Grundlagen}
\section{Was ist Router-on-a-Stick?}
Router-on-a-Stick ist ein Inter-VLAN-Routing-Verfahren,  bei dem ein Router mit nur einem physischen Interface mehrere VLANs routet. Statt vieler einzelner physischer Links wird ein gemeinsamer Trunk zwischen Switch und Router verwendet.

Der Router nutzt dafuer Subinterfaces (z.\,B.\ \texttt{Gi0/0.10} oder \texttt{Fa0/0.10}). Jedes Subinterface ist genau einem VLAN zugeordnet und erhaelt die Gateway-IP fuer dieses VLAN.

\section{VLANs, Trunking und 802.1Q}
VLANs trennen ein Layer-2-Netz logisch in mehrere Broadcast-Domaenen. Damit Frames verschiedener VLANs ueber eine gemeinsame Leitung uebertragen werden koennen, wird IEEE 802.1Q Tagging verwendet.

Zwischen Switch und Router wird ein Trunk-Port konfiguriert. Der Switch taggt die Frames je VLAN, und der Router ordnet sie ueber \texttt{encapsulation dot1Q <VLAN-ID>} den passenden Subinterfaces zu.

\section{Rolle des Default Gateways}
Endgeraete in einem VLAN senden Verkehr zu anderen Netzen an ihr Default Gateway. In dieser Uebung ist das Gateway pro VLAN jeweils die Adresse \texttt{.254} auf dem zugehoerigen Router-Subinterface.

\section{Was macht ein Subinterface am Router?}
Ein Subinterface ist ein \textbf{virtuelles Interface} auf einem physischen Router-Port. Dadurch kann ein einzelner Port (z.\,B.\ \texttt{Gi0/0} oder \texttt{Fa0/0}) mehrere logisch getrennte VLAN-Netze bedienen.

Jedes Subinterface bekommt:
\begin{itemize}
\item eine VLAN-Zuordnung via \texttt{encapsulation dot1Q <VLAN-ID>}
\item eine eigene IP-Adresse als Default Gateway des VLANs
\end{itemize}

Beispiel:
\begin{verbatim}
interface gi0/0.10 (oder fa0/0.10)
 encapsulation dot1Q 10
 ip address 10.10.10.254 255.255.255.0

interface gi0/0.20 (oder fa0/0.20)
 encapsulation dot1Q 20
 ip address 10.10.20.254 255.255.255.0
\end{verbatim}
Ohne Subinterfaces koennte der Router die VLAN-Tags am Trunk nicht sauber den jeweiligen Netzen zuordnen.

\chapter{Aufgabe: Router-on-a-Stick}
\section{Teil A: Switch konfigurieren}
\begin{enumerate}
\item VLAN 10 und VLAN 20 auf dem Switch anlegen.
\item Einen Access-Port fuer PC-VLAN10 auf VLAN 10 konfigurieren.
\item Einen Access-Port fuer PC-VLAN20 auf VLAN 20 konfigurieren.
\item Den Uplink zum Router als Trunk konfigurieren.
\item Optional: SVI-Adressen fuer VLAN 10 und VLAN 20 gemaess Tabelle setzen.
\end{enumerate}

\section{Teil B: Router konfigurieren}
\begin{enumerate}
\item Physisches Interface \texttt{Gi0/0} (oder \texttt{Fa0/0}) aktivieren.
\item Subinterface \texttt{Gi0/0.10} (oder \texttt{Fa0/0.10}) erstellen:
\begin{itemize}
\item \texttt{encapsulation dot1Q 10}
\item IP-Adresse \texttt{10.10.10.254/24}
\end{itemize}
\item Subinterface \texttt{Gi0/0.20} (oder \texttt{Fa0/0.20}) erstellen:
\begin{itemize}
\item \texttt{encapsulation dot1Q 20}
\item IP-Adresse \texttt{10.10.20.254/24}
\end{itemize}
\end{enumerate}

\section{Teil C: Endgeraete konfigurieren und testen}
\begin{enumerate}
\item IP-Adresse und Gateway auf beiden PCs gemaess Tabelle eintragen.
\item Pruefen:
\begin{itemize}
\item PC-VLAN10 pingt \texttt{10.10.10.254}
\item PC-VLAN20 pingt \texttt{10.10.20.254}
\item PC-VLAN10 pingt PC-VLAN20
\item PC-VLAN20 pingt PC-VLAN10
\end{itemize}
\item Wenn ein Ping fehlschlaegt: VLAN-Zuordnung, Trunk, Subinterfaces und Gateways kontrollieren.
\end{enumerate}
